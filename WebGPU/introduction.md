WebGPU 是新一代 Web 图形与计算 API。它让浏览器可以更直接、更高效地使用现代 GPU，既能在 Canvas 上做图形渲染，也能把通用计算任务交给 GPU 并行执行。

## WebGPU 从哪里来

在 WebGPU 之前，浏览器里最常用的 GPU 图形接口是 WebGL。WebGL 的设计理念更接近早期的 OpenGL ES：状态机式的接口易于上手，但在现代 GPU 的工作方式、性能可控性、以及“图形 + 计算”的统一能力上，都有明显的历史包袱。

随着桌面与移动端 GPU 逐步转向更“显式”的底层 API（如 Direct3D 12 / Vulkan / Metal），Web 平台也需要一个能够匹配现代 GPU 特性的标准化接口。WebGPU 正是在这样的背景下诞生的：它由 W3C 的 “GPU for the Web” 社区组推动，目标是为 Web 提供一个面向现代 GPU 的、跨平台的图形与计算标准。

你可以把 WebGPU 理解成：

- 在浏览器里可用的现代 GPU 抽象层
- 以更接近底层 API 的方式组织资源与提交命令
- 同时覆盖渲染（Render）与计算（Compute）

## WebGPU 能做什么

WebGPU 的能力可以粗略分为两类：图形渲染与 GPU 计算。

### 1. 图形渲染（Graphics Rendering）

图形渲染的目标是把三维/二维几何与材质信息，变成屏幕上的像素。WebGPU 通过渲染管线（Render Pipeline）把“输入数据 → 着色器 → 光栅化 → 输出目标”串成一条稳定的流水线，并提供缓冲区、纹理、采样器等资源类型，让你把数据高效地送进 GPU。

### 2. 在 GPU 上执行计算（GPU Computing）

GPU 不仅能画图，还非常擅长对大量数据做同一类运算。WebGPU 提供计算管线（Compute Pipeline），让你用计算着色器把图像处理、粒子模拟、物理/几何预处理、并行归约等任务放到 GPU 上执行，并把结果写回缓冲区或纹理，继续用于下一步渲染或回读到 CPU。

::: tip WebGPU 不是引擎
WebGPU 更像“工具箱”：它负责资源管理与命令提交，不会替你做相机、场景图、材质系统等高层封装。你可以在它之上构建自己的渲染框架，或使用已有引擎/库的 WebGPU 后端。
:::
