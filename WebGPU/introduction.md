# WebGPU 是什么

WebGPU 是 **Web 平台面向现代 GPU 的图形与计算 API**。

简单说，它让浏览器里的 JavaScript 能更正式地“指挥”GPU 工作：

- 要画什么
- 用什么数据来画
- 该把结果写到哪里

所以，WebGPU 不只是“在网页里画图”，它同时面向两类任务：

1. **图形渲染**：把顶点、颜色、纹理等数据变成屏幕上的像素
2. **GPU 计算**：把大量重复计算交给 GPU 并行执行

## WebGPU 能做什么

最常见的两类用途是：

- **渲染图形**：在 Canvas 上绘制 2D / 3D 内容
- **并行计算**：处理矩阵、图像、模拟、机器学习相关数据

你可以把它理解成：  
WebGPU 既能当“画图工具”，也能当“并行计算工具”。

## 开发 WebGPU 时，你在做什么

一个有一定复杂度的 WebGPU 程序，大致可以理解成：

- **大部分代码是 JavaScript**
- **小部分代码是 WGSL 着色器**

通常会经历 3 步：

1. 创建访问 GPU 所需的 JavaScript 对象
2. 创建保存数据和指令的对象
3. 把这些数据和指令发送给 GPU 执行

所以 WebGPU 的核心可以概括成一句话：

> **用 JavaScript 组织资源和命令，再把工作交给 GPU。**

## WebGPU 里最重要的几个对象

入门阶段先记住下面 4 个名字就够了：

- `navigator.gpu`：浏览器提供的 WebGPU 入口
- `GPUAdapter`：浏览器为你选择的可用 GPU 适配器
- `GPUDevice`：真正用来创建资源、编码命令的核心对象
- `GPUCanvasContext`：把 WebGPU 和 Canvas 连接起来的上下文

你可以先把它们想成一条链路：

`navigator.gpu -> GPUAdapter -> GPUDevice -> Canvas`

前面负责“拿到 GPU”，后面负责“把结果显示出来”。

## WebGPU 和 WGSL 的关系

WebGPU 不是只有 JavaScript。

- **JavaScript** 负责创建设备、缓冲区、纹理、管线和命令
- **WGSL** 负责在 GPU 上执行具体逻辑

例如：

- 顶点应该放到哪里
- 像素应该显示什么颜色
- 一组数据应该如何并行计算

所以可以把它们的分工记成：

- **WebGPU 负责调度**
- **WGSL 负责计算**

## 为什么它值得学

WebGPU 的意义不只是“新 API”。

它让 Web 应用第一次能以更统一的方式同时利用 GPU 做：

- 图形渲染
- 通用并行计算

这意味着浏览器里的应用不再只是页面和表单，它也可以承担更重的图形与计算任务。
