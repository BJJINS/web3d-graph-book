# 图形渲染流程

这一章只讲一件事：WebGPU 是如何把“顶点数据”一步步变成“屏幕像素”的。这里不放任何代码，我们介绍渲染这条流水线的概念，后续章节再把每一步落到具体 API 与案例上。

如果你是初次接触图形渲染，在看完本章后可能会感到困惑，但是没关系，下一章是一个极简的案例，我会在[绘制一个三角形](./triangleExample.md)的案例中将代码与本章节的流程关联。

## 你在做的事情是什么

一次最典型的渲染可以被概括为：

- 在 CPU 侧准备数据与渲染状态（缓冲、纹理、管线、绑定）
- 录制一段 GPU 命令（开始渲染通道、设置状态、发起绘制、结束通道）
- 让 GPU 执行命令，把结果写到一张“颜色目标纹理”
- 把这张纹理呈现到屏幕（Canvas）

从学习角度看，你可以把 WebGPU 当成“录制命令 + 提交命令”的系统：你不是每画一个像素就调用一次 API，而是描述一段工作流程，交给 GPU 批量执行。

## 流程拆解：每一步在做什么

### 1. 输入数据（Input Data）

输入数据通常由三部分构成：

- 几何数据：顶点缓冲与索引缓冲（位置、颜色、法线、纹理坐标等属性）
- 常量数据：相机矩阵、时间、材质参数等（常量缓冲 / Uniform）
- 纹理数据：贴图与采样器（决定如何从纹理取样）

这些数据本质上都是“GPU 可读的内存资源”。渲染前的很多工作，就是把 CPU 内存里的数据组织好，并传到 GPU 能访问的地方。

### 2. 顶点着色器（Vertex Shader）

顶点着色器是几何处理的入口。它对“每个顶点”执行一次，用来完成两类任务：

- 计算顶点的位置：把模型空间的点变换到裁剪空间，为后续裁剪与光栅化做准备
- 输出顶点属性：把颜色、法线、纹理坐标等写到输出里，供后续插值给片元使用

你可以把它理解为：把“离散的顶点信息”变成“图元可以理解的顶点描述”。

### 3. 图元装配（Primitive Assembly）

GPU 按你指定的拓扑方式，把顶点组装成点、线段或三角形。

- 点：一个顶点
- 线段：两个顶点
- 三角形：三个顶点

复杂模型最终都会被拆成大量三角形，这是因为三角形最稳定、最容易被光栅化。

### 4. 光栅化（Rasterization）

光栅化把“连续的几何图元”变成“离散的像素网格上的片元”。直觉上它会做两件事：

- 覆盖测试：找出三角形覆盖到了哪些像素格
- 属性插值：把顶点着色器输出的属性，按位置插值到每一个片元上

插值是理解片元着色器的关键：片元着色器并不是凭空产生输入，它的很多输入来自顶点输出的插值结果。

### 5. 片元着色器（Fragment Shader）

片元着色器对“每个片元”执行一次，用来计算最终写入颜色目标的值，例如：

- 计算颜色（基于插值后的颜色/法线/纹理坐标等）
- 采样纹理并组合材质参数
- 决定是否丢弃片元（让它不写入任何结果）

如果你把顶点着色器理解为“处理形状”，片元着色器更像是“决定外观”。

### 6. 深度/模板测试与混合（Depth/Stencil & Blending）

片元着色器输出之后，并不一定立刻写进帧缓冲，通常还会经过一组固定功能测试与混合步骤：

- 深度测试：决定哪个片元在前面，解决遮挡关系
- 模板测试：用一个额外缓冲控制哪些区域允许/禁止绘制
- 颜色混合：把“新颜色”和“已有颜色”按规则混合（例如 Alpha 混合实现半透明）

这些步骤决定了“写入的条件”和“写入的方式”，是渲染正确性与表现力的重要来源。

### 7. 输出到目标（Render Target / Pixels）

通过以上步骤，GPU 最终把结果写到一个颜色目标（通常是一张纹理）。如果这个目标与 Canvas 的交换链关联，浏览器就可以把它呈现到屏幕上，成为你看到的像素结果。

## 你需要掌握的三层模型

为了在后续章节更顺畅地阅读 API，你可以把 WebGPU 渲染理解为三层：

- 资源层：缓冲、纹理、采样器，以及它们承载的数据
- 管线层：渲染管线与绑定布局，决定“数据如何被解释、着色器如何运行”
- 命令层：渲染通道与绘制命令，决定“这一帧具体做哪些事情、按什么顺序做”

当画面出现问题时，也可以用这三层快速定位：

- 形状不对：常见在资源层（顶点数据/索引）或顶点阶段（变换）
- 外观不对：常见在片元阶段（材质/纹理采样）或混合/深度配置
- 什么都没画出来：常见在命令录制顺序、目标格式不匹配、或测试/裁剪把片元都过滤掉

## 从“像素网格”建立直觉

学习渲染流程时，一个很有帮助的直觉是：屏幕是一张像素网格，三角形在这张网格上“覆盖”并生成片元，然后每个片元计算颜色并写入。后续的示例章节会用一个非常小的交互演示，帮助你把“光栅化覆盖 + 插值 + 片元着色”这件事在脑子里彻底具体化。
